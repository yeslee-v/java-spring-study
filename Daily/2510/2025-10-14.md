# 체크 예외 vs 언체크 예외
Checked Exception
- 컴파일러가 컴파일 시점에 예외처리 강제
- 발생 가능성이 예측 가능함 → `try~catch`, `throws`로 처리
- 처리하지 않으면 컴파일 에러 발생
- `Exception`의 하위 클래스 → `SQLException`, `IOException`

Unchecked Excepetion
- 컴파일 시점에 예외 처리 강제하지 않음
- RuntimeException 클래스 상속 → `IllegalStateException`, `Error` 클래스
- 주로 개발 중에 발생하는 버그
